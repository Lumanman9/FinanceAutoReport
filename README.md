# FinanceAutoReport
The project is aiming to build an AI-powered system to automatically generate comprehensive financial reports from any type of company documents. Leverage Multi-Agent RAG architecture to analyze financials, risk, and industry data, producing structured, read-to-use reports for banks, investors and analysts. 

# Features
- Financial Report Template Extraction
- Document Understanding
- Document Chunking
- Indexing and Storage
- Multi-agent analysis
- Financial Report Generation 

# System Architecture

1. Template Setup and Data Ingestion

    **Input**: The user uploads the target Financial Report Template.

    **Action**: The system passes this through a specialized Template Extraction Module.This module uses techniques like layout analysis and optical character recognition (OCR) to understand the structure of the document (e.g., sections, headers, specific line items).

    **Output**: The module generates two distinct products:

    - A structured data definition representing the Report Template.

    - A set of generated questions dynamically formulated based on the fields identified in the template. These questions are essential for guiding the analysis required to fill out the report.

2. Document Preparation and Knowledge Base Construction

    **Input:** The user uploads all required Related Documents (e.g., internal ledgers, older reports, risk logs).

    **Document Understanding & Processing:**

    - The uploaded documents are first parsed into raw text and structured data tables.

    - Large volumes of text are then processed through a Document Chunking module, which breaks the documents down into smaller, coherent, and manageable semantic units or blocks.

    **Indexing and Storage:**

    - These text chunks are sent to an Indexing and Storage Agent.

    - This agent generates embedding vectors for each chunk and indexes them.

    - The indexed vectors, along with the original chunk text and relevant metadata, are then stored in a high-performance Knowledge Base (Vector Database) to facilitate efficient retrieval.

3. Multi-Agent Orchestration and Analysis

    **Manager Agent (The Orchestrator)**:

    - The central coordinator for the entire process.

    - It iterates through each section or field defined in the extracted template definition.

    - For a given field, it takes the associated questions (from Stage 1) and utilizes Retrieval-Augmented Generation (RAG) to query the Vector Knowledge Base for the most semantically relevant chunks of information.

    - The manager then packages the question, relevant retrieved context, and specific task instructions into a single analysis request.

    **Expert Agents (Specialized LLM Workers)**:

    The analysis request is assigned to a specialized expert agent. These agents are fine-tuned or specifically prompted to analyze content within a precise domain.

    Examples of Expert Agents:

    - Risk Expert Agent: Processes risk-related questions and generated comments, drawing on retrieved risk registers or compliance documentation.

    - Financial Statement Analysis Agent: Handles financial fields, performs calculations, and interprets financial results based on retrieved ledger data or cash flow statements.

    - Management Discussion & Analysis (MD&A) Agent: Synthesizes strategic plans and internal memos to construct management discussions.

    These agents perform their deep analysis, generate the required answers, and provide detailed comments. These responses are then returned to the Manager Agent.

4. Final Report Synthesis

    **Action:** Once all expert analyses are complete for all required template fields, a Synthesis Agent is activated.
    
    - This agent retrieves the complete structural definition of the final report template.

    - It retrieves all collected outputs (answers, calculated results, and comments) generated by the various expert agents in the previous step.

    - If necessary, it may perform one last, broad-context query against the Knowledge Base for overarching formatting rules or standard language.

    **Output:** The synthesis agent populates all fields of the report structure, ensures consistency and correct formatting throughout, and generates the final, cohesive, and comprehensive Financial Report.
